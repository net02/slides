Software Architecture
what and why?

Design carefully!

  • Sofware instabile
  • Bassa manutenibilità
  • Difficoltà di deploy
  • Incompatibilità con requisiti attuali
  • Incompatibilità con requisiti futuri

Goals and tradeoffs

Design Principles

  • Separation of concerns: divisione in feature distinte con poca sovrapposizione di funzionalità, minimizzando i punti di interazione.
  • Single Responsibility principle: ogni parte è responsabile per una sola feature, o aggregazione logica di funzionalità.
  • Principle of Least Knowledge: ogni parte non conosce i dettagli interni delle altre parti.
  • Don't repeat yourself (DRY): non duplicare funzionalità in più punti dell'architettura
  • Minimize upfront design: non complicare in previsione futura ("You ain’t gonna need it")

Microservices

Relativamente nuovo (~2010)

Microservices

Do one thing and do it well

Pro

  • ogni servizio è sostituibile
  • implementazione disomogenea (linguaggio di programmazione / database / environment)
  • architettura simmetrica anziché gerarchica (pipeline)
  • solitamente comunicano via URI

Contro

  • information barriers
  • complessità aggiuntiva (latenza di rete / load balancing / fault tolerance)
  • test & deploy

Best practices

Regole informali per migliorare la qualità del software

The first 90% of the code accounts for the first 90% of the development time. The remaining 10% of the code accounts for the other 90% of the development time.

Ninety-ninety rule, Tim Cargill

  1. Know what the code block must perform
  2. Indicate a brief description of what a variable is for (reference to commenting)
  3. Correct errors as they occur
  4. Keep your code simple
  5. Maintain naming conventions which are uniform throughout

Code smell
it works but..

  • Codice duplicato
  • Metodo troppo lungo
  • Classe troppo grande
  • Lista di parametri troppo lunga
  • Feature envy o data envy
  • Magic numbers
  • Espressioni complesse
  • What comments
  • Nomi oscuri o inconsistenti
  • Dead code
  • Generalità speculativa
